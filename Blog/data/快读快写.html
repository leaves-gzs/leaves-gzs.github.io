C++11 快读快写

高性能输入输出优化指南

For Competitive Programming

为什么需要优化?

C++ 标准输入输出流 cin/cout 在默认情况下速度较慢，主要原因是为了兼容 C 标准库而设计的同步机制带来了额外开销。

优化方法概览

  • 关闭同步和解除绑定
  • 使用 getchar / putchar 实现快读快写
  • 基于 fread / fwrite 的缓冲区优化

一、标准流优化方法

核心机制

关闭同步

ios::sync_with_stdio(false)

关闭 C++ 标准流与 C 标准流的同步，减少数据交换开销。

解除绑定

cin.tie(nullptr)

解除 cin 与 cout 的绑定，避免每次输入前刷新输出缓冲。

实现代码

#include <iostream>
using namespace std;

int main() {
    ios::sync_with_stdio(false);  // 关闭与C stdio的同步
    cin.tie(nullptr);              // 解除cin对cout的绑定
    // 你的代码
    return 0;
}

二、基于 getchar 和 putchar 的快读快写

快读实现

int read() {
    int x = 0, w = 1;
    char ch = 0;
    while (ch < '0' || ch > '9') {
        if (ch == '-') w = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * w;
}

快写实现 (非递归)

inline void write(int x) {
    static int sta[35];
    int top = 0;
    do {
        sta[top++] = x % 10;
        x /= 10;
    } while (x);
    while (top) putchar(sta[--top] + '0');
}

进阶技巧: getchar_unlocked

在单线程环境（如编程竞赛）中，可以使用 getchar_unlocked 替代 getchar 以获得更快的速度。

int readint() {
    int k = 0, f = 1, c = getchar_unlocked();
    for (; !isdigit(c); c = getchar_unlocked()) 
        if (c == '-') f = -1;
    for (; isdigit(c); c = getchar_unlocked()) 
        k = k * 10 + (c ^ 48);
    return k * f;
}

三、基于 fread 和 fwrite 的缓冲区优化

通过 fread 一次性将输入文件读入一个巨大的缓冲区，比逐个字符读入要快得多。

fread 快读

char buf[1 << 21], *p1 = buf, *p2 = buf;

inline char getc() {
    return p1 == p2 && (p2 = (p1 = buf) + fread(buf, 1, 1 << 21, stdin), p1 == p2) 
        ? EOF : *p1++;
}

inline int read() {
    int x = 0, f = 1;
    char ch = getc();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getc();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getc();
    }
    return x * f;
}

fwrite 快写

char pbuf[1 << 21], *pp = pbuf;

inline void push(const char &c) {
    if (pp - pbuf == (1 << 21)) {
        fwrite(pbuf, 1, (1 << 21), stdout);
        pp = pbuf;
    }
    *pp++ = c;
}

inline void write(int x) {
    static int sta[35];
    int top = 0;
    if (x < 0) { push('-'); x = -x; }
    do { sta[top++] = x % 10; x /= 10; } while (x);
    while (top) push(sta[--top] + '0');
    push('\n');
}

inline void flush() {
    fwrite(pbuf, 1, pp - pbuf, stdout);
    pp = pbuf;
}

核心要点总结

标准流优化

通过 ios::sync_with_stdio(false) 和 cin.tie(nullptr) 获得 3x 性能提升。

getchar 快读

比 scanf 快约 2-3 倍，实现简单，是竞赛中最常用的方法。

fread 缓冲区优化

比基础 getchar 快约 1.5 倍，适合处理大规模数据。

实践建议

根据数据规模选择

  • 数据量 < 1e5: 标准流优化
  • 数据量 1e5 - 1e6: getchar 快读
  • 数据量 > 1e6: fread 缓冲区优化

避免常见错误

  • 不要混用不同的输入输出函数
  • 使用缓冲区快写时，程序结束前必须调用 flush
  • 注意平台差异 (如 mmap 仅 Linux 支持)
