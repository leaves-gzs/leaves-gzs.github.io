图存储结构对比

核心优势

  • 空间效率高，复杂度O(n+e)
  • 缓存友好，数据连续存储
  • 插入效率高，头插法O(1)
  • 遍历高效，适合图算法
  • 实现相对简单，调试容易

数据结构定义 

const int MAXN = 100000; // 最大顶点数
const int MAXM = 200000; // 最大边数（无向图需要开双倍）

// 核心数组
int head[MAXN];    // 头节点数组
int to[MAXM];      // 终点数组
int next[MAXM];    // 下一条边数组
int weight[MAXM];  // 权值数组（可选）
int edgeCount = 0; // 当前边数

// 结构体方式（可选）
struct Edge {
    int to;
    int next;
    int weight;
};
Edge edge[MAXM]; // 边结构体数组

添加边操作 

void addEdge(int u, int v, int w) {
    to[edgeCount] = v;
    weight[edgeCount] = w;
    next[edgeCount] = head[u];
    head[u] = edgeCount++;
}

操作步骤：

  1. 存储边的终点和权值
  2. 将新边插入到链表头部
  3. 更新头指针，指向新插入的边
  4. 边计数器自增

DFS遍历实现 

bool visited[MAXN];

void dfs(int u) {
    visited[u] = true;
    
    // 遍历顶点u的所有邻接边
    for (int i = head[u]; i != -1; i = next[i]) {
        int v = to[i];
        if (!visited[v]) {
            dfs(v);
        }
    }
}

BFS遍历实现 

void bfs(int s) {
    queue<int> q;
    bool visited[MAXN] = {false};
    
    q.push(s);
    visited[s] = true;
    
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        
        // 遍历顶点u的所有邻接边
        for (int i = head[u]; i != -1; i = next[i]) {
            int v = to[i];
            if (!visited[v]) {
                visited[v] = true;
                q.push(v);
            }
        }
    }
}

常见问题与注意事项

数组大小设置错误

MAXN应大于等于最大顶点编号，MAXM应大于等于边的总数（无向图需要2倍）。

初始化遗漏

head数组必须初始化为-1，edgeCount必须重置为0，特别是在多测试用例中。

遍历顺序问题

头插法导致遍历顺序与添加顺序相反，某些算法可能需要注意这个顺序。

核心要点回顾

基本原理

用数组模拟链表的邻接表实现，通过head、to、next三个数组实现图的存储。

性能优势

缓存友好、空间高效、插入快速，特别适合大规模稀疏图的处理。

实现要点

头插法添加边，初始化head数组为-1，注意数组大小设置和多测试用例处理。
